Vue组件化

1.现实中的组件化思想体现

  • 标准

    组件要想组合到一块，肯定要有统一的标准，在软件开发中有个标准规范，不同程序员写出来的代码就会比较通用。

  • 分治

    分治就是把不同的功能封装到不同的组件中，这样不同的组件就可以独立进行生产。在软件开发中分治有利于独立的团队进行协作开发。

  • 重用

    软件开发中，代码的重用可以最直接提高代码开发效率。

  • 组合

    组件之间通过组合可以形成一款新的产品。

2.1全局组件注册语法

Vue.component(组件名称,{
	data:组件数据，
	template:组件模板内容
})

2.2组件注册基本用法

案例

三个按钮内容个为按钮被点击的次数

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="app">
    //三个子组件的数据都是独立的，每点一个按键它的数据都是相互不影响的
        <button-counter></button-counter>
        <button-counter></button-counter>
        <button-counter></button-counter>

    </div>
    <script type="text/javascript" src="vue.js"></script>
    <script>
        // 组件注册
        Vue.component('button-counter',{
            data:function(){
                return{
                    count:0
                }
            },
            //@click = "" 不能使用单引号 因为最外层使用了单引号
            template:'<button @click="handle">点击了{{count}}次</button> ',
            methods:{
                handle:function(){
                    this.count++;
                }
            }    
        })
        var vm = new Vue({
            el:"#app",
            data:{

            }
        });
    </script>
</body>
</html>

2.3组件基本使用总结

组件基本使用首先要注册，注册的方式就是通过Vue.component(）这个API来注册，参数一表示组件的名称，参数二是个对象，对象中两个比较重要的属性，data表示组件内部要的数据，template表示组件的模板内容。组件的用法就是把组件的名称放到容器里用标签把它包住，子组件也可以进行重用，复制多份多个子组件。子组件之间的数据是相互独立的。每个子组件有相互独立的数据。

2.4组件注册注意事项

1.data必须是一个函数

  • 分析函数与普通对象的对比

    例如：

data:function(){
                return{
                    count:0
                }
            }

在实例化vue实例的时候，data是一个对象，并且Vue实例本身也是一个组件，实例当中所传递的参数基本上绝大多数都是通用的，但是data的值是一个最大的不同，一个是对象一个是函数。Vue组件化里的data使用函数会形成一个闭包的环境，这样可以保证每一个组件都是拥有一份独立的数据

例如：

data: {
        uname: 'lisi',
        gender: 2,
        hobby: ['2','3'],
        // occupation: 3
        occupation: ['2','3'],
        desc: 'nihao'
      },
2.组件模板内容必须是单个根元素

所有Vue的模板内容全部都要放到el指定的容器中，因为这个容器是最外层的逻辑，子组件都是包含在容器内部的，也是一个根元素。

3.组件模板内容可以是模板字符串

  • 模板字符串需要浏览器提供支持（ES6语法）

    为什么需要模板字符串？

    通过template定义的模板，如果内容量比较大的话，都写在一行可读性是非常低的。为了使模板看起来更加直观就可以借助模板字符串

例如：

template:
            `<button @click="handle">点击了{{count}}次</button>
            <button>测试123</button>
            `,

模板内容如果比较复杂的话就可以借助模板字符串的方式来进行处理，这样模板看起来会更加直观

4.组件的命名方式

  • 短横线方式

    Vue.component('my-component'，{/*...*/})

  • 驼峰方式

    Vue.component('MyComponent',{/*...*/})

    驼峰式的命名方式如果要加到根组件里边而不是在模板里面的化就会有问题。在根组件下要换成短横线的方式

    组件注册注意事项：如果使用驼峰命名组建，那么在使用组件的时候，只能在字符串模板中用驼峰的方式使用组件，但是在普通的标签模板中，必须使用短横线的方式使用组件

2.5局部组件注册

var ComponentB = {/*...*/}

new Vue({
	el: '#app'
	components: {
	'component-a':ComponentA,

4.1 父组件向子组件传值

1.组件内部通过props接收传递过来的值
Vue.component('menu-item',{

props:['title'],

template:' <div>{{title}}</div>'

})

总结

父组件向子组件传值的规则是在子组件通过props接收父组件传过来的内容，具体是通过属性名来接收的，父组件向子组件传值的时候，属性有两种绑定方式，一种是写死的静态方式，另一种是通过动态绑定属性值进行传递。除此之外属性也可以传递多个。

3.props属性名规则

  • 在props中使用驼峰形式，模板中需要使用短横线的形式

    原因是Dom元素的属性是不区分大小写的，如果我们传递的是驼峰形式就会有问题。

  • 字符串形式的模板中没有这个限制

语法规则：

Vue.component('menu-item',{
	// 在JavaScript中是驼峰式的
	props：['menuTitle'],
	template : '<div>{{mebuTitle}}</div>'
})
<!- 在html中是短横线方式的-- >
<menu-item menu-title = "nihao"></menu-item>

4.2子组件向父组件传值

  • props传递数据原则：单向数据流（只允许父组件向子组件传递数据而不允许子组件直接操作父组件中的数据）

    如果子组件可以直接操作父组件的数据，那数据的控制逻辑就会比较复杂不容易控制。单向数据流的处理逻辑就比较清晰。Vue设计者也推荐使用单向数据流，虽然没有禁止子组件直接操作父组件的数据，但是不推荐这么做。

1.子组件通过自定义事件向父组件传递信息
<button v-on:click = '$emit("enlarge-text")'>扩大字体</button>
2.父组件监听子组件的事件
<menu-item v-on:enlarge-text = 'fontSize += 0.1'><menu-item>

案例：在子组件中点击按钮之后会控制父组件中文字大小的变化

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id = "app">
        <div :style = '{fontSize: fontSize + "px"}'>{{pmsg}}</div>
        <menu-item :parr = 'parr' @enlarge-text = 'handle'></menu-item>
    </div>
    <script src="vue.js"></script>
    <script>
        //子组件向父组件传值-基本用法
        Vue.component('menu-item',{
            props:['parr'],
            template:`
                <div>
                    <ul>
                        <li :key = 'index' v-for = '(item,index) in parr'>{{item}}</li>

                    </ul>    
                    <button @click = "parr.push('lemon')">点击</button>
                    <button @click = '$emit("enlarge-text")'>扩大父组件中字体大小</button>
                </div>
            `,
        });
        var vm = new Vue({
            el: "#app",
            data:{
                pmsg:'父组件中内容',
                parr:['apple','orange','banana'],
                fontSize:10
            },
            methods:{
                handle: function(){
                    //扩大字体大小
                    this.fontSize += 5;
                }
            }
        });
        
    </script>
</body>
</html>

效果：每次点击扩大父组件中字体大小可以增大字体

总结：子组件向父组件传递信息：首先子组件要想父组件传递一个自定义事件，同时父组件要去监听这个自定义事件。当子组件被触发的时候，会把事件抛向父组件，父组件触发之后会执行相应的动作。不应该是通过props直接去操作父组件的逻辑。因为props是单向数据流。关于单向数据流要遵循vue所规定的使用原则。

3.子组件通过自定义事件向父组件传递信息
<button v-on:click = '$emit("enlarge-text",0.1)'>扩大字体</button>

4.父组件监听子组件的事件

<menu-item v-on:enlarge-text='fontSize += $event'> </menu-item>

案例：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id = "app">
        <div :style = '{fontSize: fontSize + "px"}'>{{pmsg}}</div>
        <menu-item :parr = 'parr' @enlarge-text = 'handle($event)'></menu-item>
    </div>
    <script src="vue.js"></script>
    <script>
        //子组件向父组件传值-基本用法
        Vue.component('menu-item',{
            props:['parr'],
            template:`
                <div>
                    <ul>
                        <li :key = 'index' v-for = '(item,index) in parr'>{{item}}</li>

                    </ul> 
                    <button @click = '$emit("enlarge-text",5)'>扩大父组件中字体大小</button>  
                    <button @click = '$emit("enlarge-text",10)'>扩大父组件中字体大小</button>
                </div>
            `,
        });
        var vm = new Vue({
            el: "#app",
            data:{
                pmsg:'父组件中内容',
                parr:['apple','orange','banana'],
                fontSize:10
            },
            methods:{
                handle: function(val){
                    //扩大字体大小
                    this.fontSize += val;
                }
            }
        });

    </script>
</body>
</html>

在接收子组件传过来的值得时候父组件用**$event**。

$event就代表子组件传过来的真实的值。

总结：子组件向父组件传递信息的具体用法，首先子组件要触发事件，触发的自定义事件可以携带对应的参数，参数传递到父组件监听的处理逻辑当中。父组件如果没有封装函数，直接通过$event可以 操作传过来的值的。如果父组件绑定一个函数，那就可以$event作为参数传递给函数使用。

5.组件插槽

5.1组件插槽的作用

  • 父组件向子组件传递内容

父组件向子组件传递内容，内容为模板内容，上图中左边蓝色部分标识父组件的模板，父组件中包含了三个子组件，这三个父组件中标签是有内容的。之前标签是没有添加内容的都是空标签。现在标签加内容了，这个内容会通过插槽的方式传递给子组件，子组件模板中会预留一个位置就是插槽，子组件标签中的内容会放置在插槽中最终呈现出来。

1.插槽位置
Vue.component('alert-box',{
	template:
	`
	<div class = "demo-alert-box">
		<strong>Error!</strong>
		<slot></slot>
	</div>
	
	`,
})
2.插槽内容
<alert-box>Something bad happened.</alert-box>

插槽的位置位于子组件的模板中，所有的标签都属于vue的API，所以插槽名字是固定 的<slot>,在使用这个组件的时候通过标签的内容传递给slot最终呈现出来的结果就是父组件模板标签中的内容。

3.案例：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="app">
        <alert-box>有一个警告</alert-box>
        <alert-box></alert-box>
        <alert-box>有bug发生</alert-box>
    </div>
    <script src="vue.js"></script>
    <script>
        //组件插槽
        Vue.component('alert-box',{
            template: `
            <div>
                <strong>ERROR:</strong>    
                <slot>默认内容</slot>
                
            </div>
            `,
        });
        var vm = new Vue({
            el:'#app',

            }
        });
    </script>

</body>
</html>

ERROR: 默认内容
ERROR: 有bug发生

5.2 具名插槽用法

1.插槽定义
<div class = "container">
	<header>
		<slot name = "header"></slot>
	
	</header>
	<main>
	<slot></slot>
	</main>
	<footer>
		<slot name = "footer"></slot>
	</footer>

</div>
2.插槽内容
<base-layout>
	<h1 slot = "header">标签内容</h1>
	<p>主要内容1</p>
	<p>主要内容2</p>
	<p slot = "footer">底部内容</p>
</base-layout>

5.3 作用域插槽

  • 应用场景：父组件对子组件的内容进行加工处理
1.插槽定义
<ul>
	<li v-for:'item in list' v-bind:key = "item.id">
		<slot v-bind:item = "item">{{item.name}}</slot>
	</li>
</ul>

2.插槽内容

<fruit-list v-bind:list= "list">
	<template slot-scope = "slotProps">
		<strong v-if = "slotProps.item.current">
			{{slotProps.item.text}}
		</strong>
	</template>
</fruit-list>

案例：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <style>
        .current{
            color:orange;
        }
    </style>
    <div id="app">
        <fruit-list :list = 'list'>
            <template slot-scope ='slotProps'>
                <strong v-if = 'slotProps.info.id == 2' class ="current">{{slotProps.info.name}}</strong>
                <span v-else>{{slotProps.info.name}}</span>
            
            </template>
        </fruit-list>
    </div>
    <script src="vue.js"></script>
    <script>
        //作用域插槽
        Vue.component('fruit-list',{
            props:['list'],
            template:`
            <div>
                <li :key = 'item.id' v-for = 'item in list'>
                    <slot :info = 'item'>{{item.name}}</slot>         
                </li>
            </div>
            `,
        });
        var vm = new Vue({
            el:"#app",
            data:{
                list:[{
                    id:1,
                    name:'banana'
                },{
                    id:2,
                    name:'apple',

                },{
                    id:3,
                    name:'orange'
                }]
            },
        })
    </script>
</body>
</html>

总结：

从父组件中可以得到子组件的数据，从而对子组件的数据进行一些加工和处理，从而呈现出不同的样式。

具体的语法是首先在子组件的模板中要提供插槽，slot中间要填充我们要实现的数据，并且slot标签当中要绑定一个属性，这个属性的名字是自定义的v-bind:item = "item"，用来提供给父组件，"item"就是要操作的数据。

从父组件的角度来说要填充slot，填充的规则就是在组件标签内容的中间提供内容，内容就是通过特殊的标签template进行包裹，并且通过template关键的属性slot-scope = “XX”来得到子组件传过来的数据。 “XX”就可以得到子组件中slot绑定的数据。XX.item得到之后就可以进行数据的处理了，比如说通过一些特殊的条件进行判断。并且可以包裹一些标签。
